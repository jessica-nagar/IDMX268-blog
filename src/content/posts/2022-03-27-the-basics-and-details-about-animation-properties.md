            Many things have changed in the last few years, including the fact that website owners are increasingly building visual experiences to keep consumers on their sites. Because of our innate inclination to notice movement, developers observed that humans pay more attention to moving items.

***What is web animation?***

            Basically, it's just making things move on the internet. It is required for the creation of visually appealing websites that increase conversions and encourage customers to click, browse, and purchase items on your website. When used correctly, animations may provide valuable interaction, improve the emotional experience for users, and give your interface individuality. For producing animations, there are hundreds of libraries, tools, and plugins available, ranging from simple to complicated.

***What CSS properties can be animated?***

            Some CSS attributes are animatable, which means they can be utilized in transitions and animations. Size, color, numbers, form, percentage, and other attributes that can shift gradually from one value to another are examples. Background, background-color, border color, filter, flex, and font characteristics can all be animated.

***CSS Animations***

            We can use CSS to animate HTML components without using JavaScript. To use CSS animation, you must first define the animation's keyframes. The styles that the element will have at certain moments are stored in keyframes. CSS animations are made up of two primary components:

**Keyframes**

            They're used to show when the animation starts and stops (as well as any intermediate steps between the start and end). It is made up of three main components: animation-name which is simply the name given to the animation. Animation-stages which indicates the stages of the animation, they are usually represented as percentages. Finally animation style or CSS properties, these are the properties expected to change during the animation.

**Animation Properties**

            After you've specified the @keyframes, you'll need to add the animation attributes to make your animation work. You apply the animation attributes to the CSS selectors (or elements) you want to animate. Two properties are critical to understanding how the animation works: the animation-name, and the animation-duration. Some other properties that can be used are: animation-delay which defines when the animation will start, this uses values in seconds and milliseconds. Animation-iteration-count this specifies the number of times and animation will play on the screen. Animation-fill-mode this specifies the style for the element when it is not playing, such as before it starts, after it ends, or for both occasions. Animation-play-state this specifies if the animation is playing or if it is paused. There are two other detailed properties that I will cover separately, they are animation-timing-function and animation-direction. When animating an element, you don't have to specify all of these properties every time. Each animation attribute can be defined separately, although the animation shorthand is preferred for clearer and faster code. In the following order, all of the animation properties are applied to the same animation property: animation-name, animation-duration, animation-time-function, animation-delay, animation-iteration-count, animation-iteration-count, animation-direction, animation-fill-mode, animation-play-state.

**Animation-timing-function**

Animation-timing-function which defines the speed curve or pace of the animation. You can specify the timing with the following: ease – this is the default for animation-timing-function. The animation will slowly start and then accelerates quickly, when it reaches the halfway point it will decelerate at the same rate as it accelerated then slowly come to a stop. Linear – has a constant speed throughout the animation, people like to use this timing option for color or opacity changes. Ease-in – it starts out slow then accelerates and stops abruptly. Ease-out – it starts quickly then decelerates to a stop. Ease-in-out – this option is similar to ease but with a more subtle acceleration and deceleration.

**Animation-direction**

            This CSS property determines whether an animation should play forward, backward, or alternate between forward and backward motion. Normal – is the default for this property, this means the animation will play the keyframes forward from 0% to 100%. Reverse – will play the animation backwards from 100% to 0%. Alternate – this plays the animation forward, when it is finished it will play it backwards and repeat the process. Alternate-reverse – this one is similar to alternate, but it will play the reverse first and then repeat the process.

***Types of Basic CSS Animations***

            There are numerous types of animations that are widely utilized in websites and play a vital role in the user experience. Some of them are tooltips, hover, loading, inputs, and menus. Tooltips are text labels that appear when the user focuses on, hovers over, or touches an element. It is usually an information message that appears when a user interacts with an element in the GUI. When you move your mouse over an element, the hover pseudo-class is utilized to give it a particular effect. As a result, it has the capacity to draw users' attention to an object as soon as they hover over it. Loading animations are very important because they help to keep your user entertained while your website or page is loading. Input animations help the user to quickly fix errors or fill in missing fields when completing forms. Menus are animated elements that awe the user and keep them engaged, letting them to see all of the content on the page.

***Types of Advanced CSS Animations***

            Elements on the page can be shifted, rotated, slanted, squished, spun, and stretched with CSS animation. The benefits of CSS text animations are that your visitor’s attention will be immediately drawn to the animation typography. Adding a rainbow effect, shadow, glitch effect, and gooey effect are just a few of the various ways you may animate text on your website with CSS.

The benefits of CSS color animation is that you can have your web page background alternate between colors. CSS color animations can be used on text, buttons, borders, and other components on the website, making them excellent for attracting the visitor's attention to a specific area.

You can use slide-in animations to have an element slide in from the top, bottom, left or right of the screen to grab the viewer’ attention.

By creating a CSS rotation animation, you can rotate elements in a 2D, or 3D space. This type of animation is usually used in combination with other animation types to show an element in motion.

A CSS glow animation can contribute to the ambiance of your web site. Some developers may use it to promote an event on the site, to emphasis buttons, borders, loading animations, and other important text on their site.

Another eye-catching animation is the CSS bounce animation. To make the element’s bounce feel more natural, a few animation sub parameters, including the timing and delay function, must be adjusted while coding this animation.

Finally, the parallax scrolling animation, it’s a visual effect in which elements in the background scroll at a slower rate than those in the foreground. This gives the appearance of depth, or a “faux-3D” effect,” which is meant to make your visitor’s browsing experience more engaging.
